# Competitive Analysis Skill

## Purpose
Identify how the announcement compares to similar news in the market so we can sharpen positioning, validate USPs, and anticipate the "so what?" question from journalists.

---

## When This Skill Activates
Use this skill whenever:
- A new announcement needs competitive context before positioning is finalized
- The `/pr:competitive-scan` command is triggered
- A journalist might reasonably ask "how is this different from what [Competitor X] did?"

---

## Competitive Scan Process

### Step 1: Define the Competitive Frame
Identify what category of announcement this is. Examples:
- New product or service launch
- Partnership or contract award
- Leadership appointment
- Data/research release
- Infrastructure investment or milestone
- Policy or regulatory response

The competitive frame determines what you're scanning for. A product launch competes with other product launches in the same space. A research release competes with other data releases on the same topic.

### Step 2: Identify Comparable Announcements
Search for similar announcements from:
- Direct competitors (same industry/sector)
- Analogous organizations in adjacent sectors
- Recent news (within the last 6-12 months) on the same theme

For each comparable, note:
- Organization name
- What they announced
- When (date)
- Key claims or positioning they used
- Media coverage it received (if findable)
- Estimated impact/scale

### Step 3: Identify Differentiation Gaps
Compare our announcement against the competitive set across these dimensions:

| Dimension | Our Announcement | Competitor Average | Our Advantage? |
|-----------|-----------------|-------------------|----------------|
| Scale/magnitude | | | |
| Novelty | | | |
| Timing | | | |
| Specificity of claims | | | |
| Supporting proof points | | | |
| Human interest angle | | | |

### Step 4: Sharpen the USP
Based on the gap analysis, articulate:
- **Our strongest differentiator**: What we can say that no one else can say right now
- **Claims to avoid**: Anything a competitor has already claimed that would make us sound derivative
- **Angles to exploit**: Dimensions where we are clearly ahead that may not be obvious in the release draft

### Step 5: Anticipate Journalist Skepticism
For each likely USP, generate the "yes, but..." objection a skeptical journalist would raise, and a one-sentence response.

---

## Output Format
Produce a structured competitive brief with:
1. Competitive landscape summary (3-5 sentences)
2. Comparison table (as above)
3. Recommended primary USP with rationale
2. Supporting USPs (2-3)
3. Proof points that validate each USP
4. Anticipated objections and responses
5. Positioning statement (1-2 sentences suitable for use in pitch copy)

---

## Important Caveats
- Do not overstate differentiation. If competitors have made similar claims, acknowledge it and look harder for genuine distinction.
- Competitive intelligence from web search is only as current as available public information. Flag if key competitors have limited public presence.
- The goal is *informed* positioning, not spin.
